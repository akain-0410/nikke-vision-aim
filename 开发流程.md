# MarbleAim 开发流程

这份文档记录项目从需求拆解、图像识别、轨迹建模到本地构建的完整流程，便于以后修改和排查问题。本衍生版本采用 AI-first 开发流程：维护者负责需求、实机素材、测试和验收，代码分析与实现主要由 AI 编程工具完成。

## 1. 项目目标

程序只做三件事：

1. 截取目标游戏窗口。
2. 识别碰撞外框、方块和瞄准线。
3. 用数学模型计算轨迹，并通过透明覆盖层显示。

程序不读取游戏内存，不控制鼠标，也不会自动发射。

## 2. 目录结构

```text
marble_aim/
  __main__.py     启动入口、选择窗口、加载配置
  app.py          主状态机和各模块调度
  capture.py      Windows 窗口查找、客户区坐标、截屏、热键
  vision.py       白框、方块和瞄准线识别
  geometry.py     连续碰撞检测、反射和轨迹模拟
  overlay.py      透明覆盖层、控制按钮和设置窗口
  calibration.py  弹珠运动跟踪与参数拟合
  config.py       配置数据结构和 JSON 读写
  number_recognition.py  高级模式数字 OCR
  assets/
    digit_templates.npz  砖块 HP 与弹珠数量模板
tests/            自动化测试
tools/            静态图片和录屏分析工具
```

## 3. 主运行流程

程序约每 33ms 执行一次 `ApplicationController.tick()`：

```text
截取窗口
  ↓
判断窗口尺寸是否变化
  ↓
读取瞄准线状态
  ├─ 初次出现 → 识别外框和方块，锁定场景
  ├─ 持续出现 → 只快速更新当前方向和数学轨迹
  ├─ 连续消失 600ms → 判定小球飞行，保持锁定场景
  └─ 飞行后连续出现 600ms
       ↓
     暂时隐藏覆盖层，取得干净画面
       ↓
     比较绝对网格中的方块布局
       ├─ 未变化 → 保持原场景
       └─ 已变化 → 只更新方块和发射点，外框保持不变
```

“重启识别”按钮与自动换回合不同。按钮会清空所有识别缓存，重新识别外框、方块、球半径和发射点，相当于重启识别模块。

## 4. 窗口查找与坐标

程序不需要游戏安装路径。`capture.py` 使用 Win32 枚举当前可见窗口，并按窗口标题匹配。

- 首次启动：用户从列表选择窗口。
- 后续启动：从 `%LOCALAPPDATA%\MarbleAim\config.json` 读取标题。
- 标题不存在：重新显示窗口选择列表。

截屏、识别和碰撞计算统一使用游戏客户区像素坐标。窗口移动后只更新覆盖窗口在桌面上的位置，场景内部坐标不变。窗口尺寸变化后，所有比例重新计算。

## 5. 白色碰撞框识别

16:9 全屏画面优先使用从 3840×2160 实机数据等比缩放的固定区域，不逐帧重新检测边框。非 16:9 窗口才调用 `vision.py::_detect_collision_frame()`：

1. 在 HSV 中提取低饱和、高亮度像素。
2. 统计每一列的浅色像素数量，寻找左右竖边。
3. 统计每一行的浅色像素数量，寻找上下横边。
4. 优先选择最靠近水池的单像素内沿，而不是宽的装饰外沿。

初次锁定后，自动换回合不会重新检测外框，避免动画或覆盖层造成边界跳动。只有窗口尺寸变化或点击“重启识别”才重新建立区域。

物理模型只把上、左、右视为反弹墙；底边是开放出口。

## 6. 方块识别

方块不是逐个自由轮廓定位，而是白框内的六列规则网格：

1. 根据白框宽度计算基础格距：`step = board.width / 6`。
2. 从可靠彩色轮廓估计实际方块边长和网格相位。
3. 将所有候选块吸附到整数行、列。
4. 对水色方块使用边缘密度和深色像素密度补充判断。
5. 用多帧中位数消除立绘上下浮动的 1–2 像素误差。

换回合签名使用相对白框的绝对行列，因此“所有旧块整体下移一行”也会被识别为场景变化。

标准模式不会运行数字 OCR，方块内部数字不参与规划。只有高级模式开启时，`number_recognition.py` 才会加载模板并识别砖块 HP 和 `*XX` 弹珠数量。无法可靠识别的砖块会保守地视为本轮不会消失。

## 7. 瞄准线识别

瞄准线有两个通道：

### 完整通道

- 提取浅黄色/白色虚线。
- 使用概率 Hough 找候选直线。
- 收集候选附近所有虚线像素。
- 使用 `cv2.fitLine` 做亚像素方向拟合。
- 将直线与白框底边求交，得到本次发射点。

箭头中轴用于虚线太短、角度太小时的回退和硬门控。没有箭头时，不接受飞行弹珠、特效或水纹产生的假直线。

### 快速通道

鼠标移动时使用：

```text
相邻帧差分区域 ∩ 当前浅色像素
```

只在白框附近的裁剪区域运行，减少计算量。方向经过短窗口中值、死区和滞回处理，避免多次反弹后放大一像素抖动。

瞄准线连续消失和重新出现都需要经过 `aim_transition_delay_ms` 防抖，默认 600ms。

## 8. 球半径

游戏瞄准圈比实际飞行弹珠大，不能直接作为物理半径。实机截图测得 4K 全屏下弹珠直径约 `62px`，因此优先按方块边长 `0.202` 计算半径，无方块时按白框宽度 `0.032` 回退。

窗口从 1080p 调整到 2K 或 4K 后，球半径、网格、运动差分核、线宽和碰撞距离会同比缩放。

## 9. 轨迹数学模型

弹珠是半径为 `r` 的圆，轨迹点表示球心。

- 墙壁向水池内部收缩 `r`。
- 方块碰撞盒向外扩张 `r`。
- 球心沿射线进行连续碰撞检测，避免高速穿透。
- 碰撞后按法线反射：

```text
v' = v - 2(v·n)n
```

方块本体按直角正方形处理。方块扩张后，球心绕尖角运动时自然形成半径为 `r` 的圆弧。

为防止角落死循环，每次碰撞后沿新方向移动一个很小的 epsilon，并限制最大碰撞次数和最大总距离。

推荐搜索只覆盖正前方不到 180 度的可发射角度，并使用粗搜索加局部精搜索。第 2–7 行的危险权重逐层指数增加，第 7 行临界砖块优先级最高。

高级模式会按弹珠发射顺序扣除 HP。砖块归零后从后续弹珠的活动场景中移除，并重新计算剩余路径。评分同时考虑有效伤害、击破数量和能否清空全部临界行。

## 10. 覆盖层

覆盖层使用 PySide6：

- 透明、置顶。
- 主覆盖层鼠标穿透。
- 紫色显示当前预测轨迹。
- 绿色准星显示最佳推荐路线的首次碰撞点。
- 模式 C 和 D 显示绿色完整推荐轨迹。
- 圆形显示碰撞时的球体范围。
- 绿色虚框显示锁定方块。
- 青色显示上、左、右碰撞边。
- 橙色虚线显示开放底边。

独立控制条不穿透鼠标：

- `重启识别`：完整清空并重新识别。
- `选择窗口`：关闭旧截屏对象并切换到另一个可见窗口。
- `模式 A/B/C/D`：切换绿色最佳落点、绿色推荐轨迹和紫色当前轨迹的组合。
- `高级 OFF/ON`：切换 HP OCR、弹珠数量识别和多球动态消块模拟。
- `退出助手`：停止定时器、释放截屏资源、保存配置并结束进程。

## 11. 配置

普通用户配置：

```text
%LOCALAPPDATA%\MarbleAim\config.json
```

仓库只提交 `config.example.json`，不提交个人 `config.json`。

重要参数：

- `vision.detection_interval_ms`：完整识别间隔。
- `vision.aim_transition_delay_ms`：线消失/重现防抖时间。
- `physics.max_collisions`：最大碰撞次数。
- `physics.reflection_bias_deg`：反射角修正。
- `physics.volley_count`：弹珠数量识别失败时的回退值。
- `physics.advanced_simulation`：是否启用数字 OCR 和多球动态消块模拟。
- `calibration.boundary_offsets`：四边边界偏移。

## 12. 本地开发

创建环境：

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -e ".[test]"
```

启动：

```powershell
.\.venv\Scripts\python.exe -m marble_aim
```

调试识别：

```powershell
.\.venv\Scripts\python.exe -m marble_aim --debug-view
```

运行测试：

```powershell
.\.venv\Scripts\python.exe -m pytest
```

每次改识别逻辑时至少验证：

1. 初始白框和方块数量。
2. 小角度瞄准线。
3. 发射后没有假预测线。
4. 新瞄准线出现后正确换场。
5. 窗口缩放后的对齐。
6. 球外缘接触墙和方块时才反弹。
7. 标准模式不会加载数字 OCR。
8. 高级模式正确识别 HP 和弹珠数量。
9. 砖块归零后剩余弹珠会重新计算路线。
10. 第 2–7 行危险权重严格递增。

## 13. Git 工作流

查看修改：

```powershell
git status
git diff
```

运行测试后提交：

```powershell
git add <本次修改的具体文件>
git commit -m "描述本次修改"
git push
```

不要提交：

- `.venv/`
- `build/`、`dist/`
- 个人 `config.json`
- 视频、截图和调试输出
- `__pycache__/`、`.pytest_cache/`

这些规则已经写入 `.gitignore`。

## 14. 打包流程

确认源码版实机验证无误后再打包：

```powershell
.\.venv\Scripts\python.exe -m pip install -e ".[package]"
.\.venv\Scripts\python.exe -m PyInstaller --noconfirm --clean MarbleAim.spec
```

输出：

```text
dist\MarbleAim.exe
```

当前使用单文件模式。它便于分享，但首次启动需要解压依赖，会比文件夹模式慢。

原项目尚未声明许可证时，构建产物仅用于本地测试，不应公开分发。许可状态明确后，公开分发前建议：

1. 在干净的 Windows 10/11 电脑测试。
2. 扫描病毒并提供 SHA256。
3. 使用代码签名降低 SmartScreen 警告。
4. 标注版本号和更新日志。

## 15. 录屏预测对比

开发调试时，可以把指定角度的单条预测路径叠加到录屏。示例参数必须根据具体素材调整：

```powershell
.\.venv\Scripts\python.exe tools\overlay_video_prediction.py `
  "input.mp4" "prediction.mp4" `
  --aim-frame <松手前帧号> --start-time <开始秒数> --end-time <结束秒数> `
  --ball-radius <素材中的球半径> --max-collisions 8
```

脚本会使用末帧之前的多帧中位数稳定角度，并生成同名 `.json` 参数记录和 `.preview.png` 预览图。
