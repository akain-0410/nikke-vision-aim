# MarbleAim V6.0.0 使用说明

## 当前发布状态

当前仓库只提供源码。由于上游项目尚未声明许可证，公开 EXE 和其他二进制分发已暂停；待许可状态明确后再恢复 Release 附件。

## 源码运行

要求 64 位 Python 3.12：

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install --upgrade pip
.\.venv\Scripts\python.exe -m pip install -e ".[test]"
.\.venv\Scripts\python.exe -m marble_aim
```

首次启动时从列表选择游戏窗口。程序会记住窗口标题；目标窗口不存在时会重新显示选择列表。

## 基本操作

1. 先启动游戏，并保持窗口可见。
2. 启动 MarbleAim 并选择游戏窗口。
3. 使用“模式 A/B/C/D”切换最佳落点和轨迹组合。
4. 默认保持“高级 OFF”；需要 HP 和多球动态消块模拟时再开启。
5. 调整窗口尺寸后等待自动重新识别，或点击“重启识别”。
6. 使用“退出助手”保存配置并结束程序。

个人配置保存在：

```text
%LOCALAPPDATA%\MarbleAim\config.json
```

删除该文件可以恢复默认设置并重新选择窗口。

## 分辨率

固定全屏区域以 3840×2160 的 16:9 实机画面为基准，其他 16:9 分辨率按比例缩放。非 16:9 窗口使用视觉边框检测作为回退。

建议：

- 使用 Windows 推荐的显示缩放。
- 尽量不要让游戏窗口跨越两个不同 DPI 的显示器。
- 独占全屏无法捕获时，切换到窗口化或无边框窗口模式。
- 4K 和高级模拟会提高截屏、OCR 与路径搜索负载。

高级模式开启后，程序会识别砖块 HP 和弹珠数量。砖块被击破后，剩余弹珠会基于更新后的场景重新计算路线。数字识别失败的砖块会保守地视为本轮不会消失。

标准模式和高级模式都会使用第 2–7 行生存权重；越接近第 8 行失败区，砖块优先级越高。高级模式会进一步考虑有效伤害、能否击破和是否清空临界行。

## 本地构建

以下步骤仅用于本地测试。在上游许可状态明确前，请勿公开分发构建产物。

双击 `build_release.bat`，或执行：

```powershell
.\.venv\Scripts\python.exe -m pip install -e ".[package]"
.\.venv\Scripts\python.exe -m PyInstaller --noconfirm --clean MarbleAim.spec
```

输出文件：

```text
dist\MarbleAim.exe
```

单文件版本首次启动时需要把依赖解压到临时目录，因此会比源码或文件夹版本稍慢。
